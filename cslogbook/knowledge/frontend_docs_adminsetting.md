# การตั้งค่าระบบฝึกงาน CSLogbook (สำหรับผู้ดูแลระบบ)

## หน้าจอหลักสำหรับการตั้งค่า (Admin Settings Interface)

## 1. การตั้งค่าหลักสูตร (Curriculum Settings)


*   **เจ้าหน้าที่ภาควิชาจะทำอะไรได้บ้าง?:**
    *   **จัดการข้อมูลหลักสูตร:** เจ้าหน้าที่ภาควิชาสามารถเพิ่มหลักสูตรใหม่ แก้ไขข้อมูลหลักสูตรที่มีอยู่ หรือลบหลักสูตรที่ไม่ใช้งานแล้วได้
    *   **กำหนดเงื่อนไขหน่วยกิตสำหรับการฝึกงานและโครงงานต่อหลักสูตร:**
        * กำหนดจำนวนหน่วยกิตสะสมขั้นต่ำที่นักศึกษาในหลักสูตรนั้นๆ จะต้องมีเพื่อสามารถยื่นคำร้องขอฝึกงานได้
        * กำหนดจำนวนหน่วยกิตสะสมขั้นต่ำสำหรับการฝึกงาน (Internship)
        * กำหนดจำนวนหน่วยกิตวิชาภาค/จำนวนหน่วยกิตสะสมขั้นต่ำสำหรับโครงงานพิเศษ (Special Project)
    *   **กำหนดสถานะการใช้งานหลักสูตร:** สามารถตั้งค่าได้ว่าหลักสูตรไหนกำลังเปิดใช้งาน (Active) หรือปิดใช้งาน (Inactive) ซึ่งจะมีผลต่อการแสดงผลให้นักศึกษาเลือก และใช้ในการตรวจสอบคุณสมบัติ
*   **การตั้งค่านี้มีผลอย่างไรกับระบบ?:**
    *   การตั้งค่าส่วนนี้สำคัญ เพราะระบบจะใช้ข้อมูลนี้ในการตรวจสอบคุณสมบัติเบื้องต้นของนักศึกษาเมื่อต้องการยื่นคำร้องขอฝึกงาน (คพ.05) หรือขอทำโครงงานพิเศษ ระบบจะกรองและอนุญาตเฉพาะนักศึกษาที่มีคุณสมบัติตรงตามเกณฑ์ที่เจ้าหน้าที่ภาควิชากำหนดไว้สำหรับแต่ละหลักสูตร

## 2. การตั้งค่าปีการศึกษาและภาคเรียน (Academic Year/Semester Settings)

ส่วนนี้จะช่วยให้เจ้าหน้าที่ภาควิชาสามารถกำหนดปีการศึกษา ภาคเรียนปัจจุบัน และช่วงเวลาสำคัญต่างๆ ที่เกี่ยวข้องกับการลงทะเบียนฝึกงานและโครงงานพิเศษได้

*   **หน้าจอที่เกี่ยวข้องในระบบคืออะไร?:**
    *   หน้าจอสำหรับการจัดการตั้งค่าปีการศึกษาและภาคเรียน 
    *   ฟังก์ชันเสริมสำหรับช่วยในการตรวจสอบและจัดการข้อมูล 
*   **เจ้าหน้าที่ภาควิชาจะทำอะไรได้บ้าง?:**
    *   **กำหนดปีการศึกษาและภาคเรียนปัจจุบัน:** ตั้งค่าปีการศึกษาและภาคเรียนที่ระบบกำลังดำเนินการอยู่ เพื่อให้ข้อมูลและการดำเนินการต่างๆ สอดคล้องกับช่วงเวลาปัจจุบัน
    *   **กำหนดช่วงเวลาของแต่ละภาคเรียน:** ระบุวันที่เริ่มต้นและวันที่สิ้นสุดของแต่ละภาคเรียนได้
    *   **กำหนดช่วงเวลาการลงทะเบียนฝึกงาน:**
        *   วันที่ระบบเริ่มเปิดรับคำร้องขอฝึกงาน (คพ.05) จากนักศึกษา
        *   วันที่ระบบสิ้นสุดการเปิดรับคำร้องขอฝึกงาน
    *   **กำหนดช่วงเวลาการลงทะเบียนโครงงานพิเศษ:** (มีลักษณะคล้ายกับการฝึกงาน)
        *   วันที่เริ่มเปิดรับคำร้องขอทำโครงงานพิเศษ
        *   วันที่สิ้นสุดการเปิดรับคำร้องขอทำโครงงานพิเศษ

## 3. การตั้งค่าเกณฑ์คุณสมบัติการฝึกงานและโครงงาน (Internship & Project Eligibility Settings)

ในส่วนนี้ เจ้าหน้าที่ภาควิชาจะสามารถกำหนดเงื่อนไขและเกณฑ์คุณสมบัติที่ละเอียดมากขึ้นสำหรับการเข้าร่วมการฝึกงานหรือการทำโครงงานพิเศษ นอกเหนือจากเงื่อนไขที่กำหนดไว้ในระดับหลักสูตรได้

*   **หน้าจอที่เกี่ยวข้องในระบบคืออะไร?:**
    *   ส่วนจัดการ UI และ Logic หลักในการกำหนดเกณฑ์คุณสมบัติเหล่านี้
*   **เจ้าหน้าที่ภาควิชาจะตั้งค่าอะไรได้บ้างสำหรับการฝึกงาน (Internship Eligibility)?:**
    *   ชั้นปีการศึกษาขั้นต่ำที่นักศึกษาสามารถขอฝึกงานได้ (เช่น กำหนดเป็นปี 3 ขึ้นไป)
    *   จำนวนหน่วยกิตสะสมขั้นต่ำโดยรวมที่นักศึกษาต้องมี (อาจเป็นเกณฑ์เพิ่มเติมจากที่กำหนดในระดับหลักสูตร)
    *    จำนวนชั่วโมงการฝึกงานขั้นต่ำที่นักศึกษาจะต้องปฏิบัติให้ครบถ้วนตามข้อกำหนด
    *   จำนวนวันฝึกงานขั้นต่ำที่นักศึกษาจะต้องปฏิบัติ
*   **เจ้าหน้าที่ภาควิชาจะตั้งค่าอะไรได้บ้างสำหรับโครงงานพิเศษ (Project Eligibility)?:**
    *   ชั้นปีการศึกษาขั้นต่ำสำหรับนักศึกษาที่ต้องการทำโครงงานพิเศษ
    *   จำนวนหน่วยกิตสะสมรวมขั้นต่ำ
    *   จำนวนหน่วยกิตเฉพาะวิชาเอก/วิชาภาคขั้นต่ำ
    *   (Checkbox/Switch) ตัวเลือกสำหรับกำหนดว่านักศึกษาจำเป็นต้องผ่านการฝึกงานมาก่อนหรือไม่ จึงจะมีสิทธิ์ในการทำโครงงานพิเศษ
*   **การตั้งค่านี้มีผลอย่างไรกับระบบ?:**
    *   ระบบจะใช้เกณฑ์เหล่านี้ร่วมกับข้อมูลหลักสูตรและสถานะปัจจุบันของนักศึกษาในการตรวจสอบคุณสมบัติอย่างละเอียด เพื่อให้มั่นใจว่านักศึกษาที่เข้าร่วมโครงการฝึกงานหรือโครงงานพิเศษมีคุณสมบัติตรงตามข้อกำหนดของภาควิชาที่ท่านตั้งไว้

## 4. การตั้งค่าการแจ้งเตือน (Notification Settings)

ส่วนนี้ช่วยให้เจ้าหน้าที่ภาควิชาสามารถกำหนดค่าการแจ้งเตือนอัตโนมัติไปยังผู้ใช้ที่เกี่ยวข้อง (นักศึกษา, อาจารย์, พี่เลี้ยง) เพื่อให้กระบวนการต่างๆ เป็นไปอย่างราบรื่น

*   **เจ้าหน้าที่ภาควิชาจะทำอะไรได้บ้าง?:**
    *   **เปิด/ปิด การแจ้งเตือนผ่านช่องทางต่างๆ:** เช่น Email, การแจ้งเตือนภายในระบบ (In-App System Notification)
    *   **กำหนดประเภทการแจ้งเตือนที่ต้องการเปิดใช้งาน:**
        *   การแจ้งเตือนเมื่อมีการเปลี่ยนแปลงสถานะของเอกสาร (เช่น คพ.05 ได้รับการอนุมัติ/ไม่อนุมัติ)
        *   การแจ้งเตือนเมื่อใกล้ถึงกำหนดเวลาส่งเอกสารสำคัญ
        *   การแจ้งเตือนเมื่อมีรายการที่เจ้าหน้าที่ภาควิชา (ผู้ดูแลระบบ) หรืออาจารย์ที่ปรึกษาจำเป็นต้องดำเนินการ (เช่น มีคำร้อง คพ.05 ใหม่รอการอนุมัติ)
    *   **กำหนดระยะเวลาการแจ้งเตือนล่วงหน้า:**
        *   จำนวนวันล่วงหน้าก่อนถึงกำหนดส่งเอกสาร ที่ระบบจะส่งการแจ้งเตือนไปยังนักศึกษา
        *   `reviewDueDays`: จำนวนวันที่ควรใช้ในการตรวจเอกสาร (อาจใช้ในการแจ้งเตือนอาจารย์ที่ปรึกษาหากมีเอกสารค้างการตรวจสอบนานเกินไป)
*   **การตั้งค่านี้มีผลอย่างไรกับระบบ?:**
    *   ช่วยให้กระบวนการฝึกงานและโครงงานเป็นไปอย่างราบรื่นยิ่งขึ้น โดยมีการสื่อสารไปยังผู้เกี่ยวข้องอย่างอัตโนมัติและทันท่วงที ลดโอกาสในการพลาดกำหนดเวลาหรือการดำเนินการที่สำคัญ

## 5. การตั้งค่าสถานะนักศึกษา (Student Status Settings)

เจ้าหน้าที่ภาควิชาสามารถกำหนดและจัดการสถานะต่างๆ ของนักศึกษา ซึ่งอาจมีผลต่อสิทธิ์และการเข้าถึงฟังก์ชันต่างๆ ในระบบ

*   **หน้าจอที่เกี่ยวข้องในระบบคืออะไร?:**
    *   หน้า UI สำหรับการจัดการการตั้งค่าสถานะนักศึกษา
*   **เจ้าหน้าที่ภาควิชาจะทำอะไรได้บ้าง?:**
    *   **จัดการสถานะนักศึกษา:** เจ้าหน้าที่ภาควิชาสามารถกำหนดสถานะต่างๆ ของนักศึกษา (เช่น ปกติ, กำลังฝึกงาน, กำลังทำโครงงาน, ปีตกค้าง)
    *   **กำหนดคุณลักษณะของสถานะ:**
        *   กำหนดสีที่ใช้แทนแต่ละสถานะเพื่อการแสดงผลที่ชัดเจน
        *   เพิ่มคำอธิบายสำหรับแต่ละสถานะ
    *   **กำหนดเงื่อนไขที่เกี่ยวข้องกับสถานะ:**
        *   เช่น กำหนดจำนวนปีการศึกษาสูงสุดสำหรับนักศึกษาที่อยู่ในสถานะ "ตกค้าง"
*   **การตั้งค่านี้มีผลอย่างไรกับระบบ?:**
    *   สถานะของนักศึกษามีผลโดยตรงต่อการตรวจสอบคุณสมบัติในการเข้าร่วมการฝึกงานและโครงงาน
    *   นักศึกษาที่อยู่ในสถานะ "กำลังฝึกงาน" อาจมีหน้าจอการใช้งาน (User Interface - UI) หรือกระบวนการทำงาน (Workflow) ที่แตกต่างออกไปจากนักศึกษาในสถานะอื่น


## การตั้งค่าเหล่านี้เชื่อมโยงกับส่วนอื่นอย่างไร?

*   **การจัดการสิทธิ์การเข้าถึง (Permissions Management):**
    *   เจ้าหน้าที่ภาควิชาสามารถกำหนดได้ว่าบทบาทผู้ใช้ใด (เช่น อาจารย์ที่ปรึกษา, เจ้าหน้าที่ภาควิชา) สามารถเข้าถึงหรือแก้ไขข้อมูลส่วนใดได้บ้างที่เกี่ยวข้องกับระบบการฝึกงาน
*   **การดูรายงาน (Reporting):**
    *   ข้อมูลจากการตั้งค่าต่างๆ (เช่น จำนวนนักศึกษาตามหลักสูตร, สถานะการอนุมัติเอกสาร) จะเป็นข้อมูลพื้นฐานสำหรับการสร้างรายงานสรุปผลการดำเนินงานของโครงการฝึกงาน
    *   เจ้าหน้าที่ภาควิชาสามารถเข้าดูรายงานเหล่านี้ได้จากส่วนการจัดการรายงาน
