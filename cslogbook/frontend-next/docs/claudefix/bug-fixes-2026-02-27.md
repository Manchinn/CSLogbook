## Bug #1 — Login Error Displays Raw JSON Instead of Message

### Severity
`Medium` — All users see raw JSON on failed login attempts

### File
`cslogbook/frontend-next/src/lib/api/client.ts`

### Description
When the backend returns an error (e.g. wrong credentials), the error message rendered to the user was the full raw JSON string:

```
{"success":false,"message":"ชื่อผู้ใช้หรือหัสผ่านไม่ถูกต้อง"}
```

instead of just the clean message:

```
ชื่อผู้ใช้หรือหัสผ่านไม่ถูกต้อง
```

### Root Cause
The `try/catch` block in `apiFetch()` was catching its own `throw new Error(parsed.message)`, causing the outer `catch` to re-throw using the raw `errorBody` string:

```ts
// Before (broken)
try {
  const parsed = JSON.parse(errorBody) as { message?: string };
  throw new Error(parsed.message ?? "API request failed"); // ← caught by own catch ↓
} catch {
  throw new Error(errorBody || "API request failed"); // ← uses raw JSON string
}
```

### Fix
Separate JSON parsing from the error throw so only the `JSON.parse()` call is inside the try/catch:

```ts
// After (fixed)
let message = "API request failed";
try {
  const parsed = JSON.parse(errorBody) as { message?: string };
  message = parsed.message ?? message;
} catch {
  message = errorBody || message;
}
throw new Error(message);
```

### Impact
All API error responses across the entire app use `apiFetch`, so this fix improves error messaging globally — not just on the login page.

## Bug #2 — Dashboard Internship Status Badges Show Raw Enum Values

### Severity
`Low` — Visual regression; data is correct but labels are unreadable

### File
`cslogbook/frontend-next/src/components/dashboard/StudentInternshipStatusWidget.tsx`

### Description
The internship status card on the Student Dashboard displayed raw snake_case database enum values in the status badges:

| Badge | Before | After |
|---|---|---|
| สถานะ | `not_requested` | `ยังไม่ร้องขอ` |
| ใบรับรอง | `not_requested` | `ยังไม่ร้องขอ` |

### Root Cause
Two issues combined:

1. `statusChip` was set directly from `certificate?.status` with no label mapping.
2. The certificate badge used a truthy check that failed to catch `"not_requested"`:
   ```tsx
   // Before (broken) — "not_requested" is truthy, so it renders the raw value
   ใบรับรอง: {certificate?.status ? certificate.status : "ยังไม่ร้องขอ"}
   ```

### Fix
Added a `STATUS_LABELS` map and a `labelStatus()` helper function, then applied it to both badges:

```ts
const STATUS_LABELS: Record<string, string> = {
  not_requested: "ยังไม่ร้องขอ",
  pending:       "รอดำเนินการ",
  approved:      "อนุมัติแล้ว",
  rejected:      "ไม่อนุมัติ",
  ready:         "พร้อมใช้งาน",
  active:        "กำลังฝึกงาน",
  completed:     "เสร็จสิ้น",
};

function labelStatus(value?: string | null, fallback = "-") {
  if (!value) return fallback;
  return STATUS_LABELS[value] ?? value;
}
```

```tsx
// After (fixed)
<span>สถานะ: {labelStatus(statusChip)}</span>
<span>ใบรับรอง: {labelStatus(certificate?.status, "ยังไม่ร้องขอ")}</span>
```

## Bug #4 — Teacher Meeting Approvals Page Renders Blank Content

### Severity
`High` — Teacher page completely non-functional; no content, no empty state, no error

### File
`cslogbook/frontend-next/src/lib/services/teacherService.ts`

### Description
The `/teacher/meeting-approvals` page rendered only the heading and description but showed no content, no loading indicator, no empty state, and no error message. The `TeacherEmptyState` that should display "ไม่มีบันทึกการพบที่รออนุมัติในขณะนี้" never appeared.

### Root Cause
A mismatch between the TypeScript type and the actual API response shape:

```
API returns: { success: true, data: { items: [], summary: {}, meta: {}, teacher: {} } }
Service typed:  apiFetchData<MeetingLogApproval[]>  →  data = { items: [], ... }  (object, not array)
Service returns: data || []  →  returns the whole object
```

The page code then checks `meetings.length === 0` which is `undefined` on a plain object — so neither the empty state condition (`=== 0`) nor the table condition (`> 0`) evaluated to `true`, leaving the content area blank.

### Fix
Update the type parameter and extract `items`:

```ts
// Before
const data = await apiFetchData<MeetingLogApproval[]>(url, { method: "GET", token });
return data || [];

// After
const data = await apiFetchData<{ items: MeetingLogApproval[] }>(url, { method: "GET", token });
return data?.items || [];
```
