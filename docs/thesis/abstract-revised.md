# บทคัดย่อ (ฉบับแก้ไขตามคำแนะนำกรรมการ)

> **โครงสร้างตามที่กรรมการแนะนำ:**
> 1. สรุปงานที่ทำ (ปัญหา + วัตถุประสงค์)
> 2. พัฒนาด้วยอะไรบ้าง (เทคโนโลยี)
> 3. ผลการพัฒนา (ฟีเจอร์ที่ได้)
> 4. การนำไปใช้งาน (deploy + ผลทดสอบ)

---

## ฉบับแนะนำ (copy ไปวางใน Google Docs แทนบทคัดย่อเดิม)

ปริญญานิพนธ์นี้นำเสนอการพัฒนาระบบสมุดบันทึกการฝึกงานและติดตามความคืบหน้าปริญญานิพนธ์สำหรับนักศึกษาภาควิชาวิทยาการคอมพิวเตอร์และสารสนเทศ (CS Logbook) เพื่อแก้ไขปัญหาการจัดการเอกสารและกระบวนการทางวิชาการที่ยังพึ่งพาเอกสารกระดาษเป็นหลัก ซึ่งก่อให้เกิดความล่าช้า ข้อผิดพลาด และภาระงานแก่ทั้งนักศึกษา อาจารย์ที่ปรึกษา และเจ้าหน้าที่ภาควิชา โดยระบบได้อ้างอิงข้อมูลตามหลักสูตรวิทยาศาสตรบัณฑิต สาขาวิชาวิทยาการคอมพิวเตอร์ (หลักสูตรปรับปรุง พ.ศ. 2564) สำหรับนักศึกษาปีการศึกษา 2567 และ 2568

ระบบพัฒนาในรูปแบบเว็บแอปพลิเคชันด้วย Next.js 16 (App Router), React 19 และ TypeScript สำหรับส่วนหน้า (Frontend) โดยใช้ TanStack Query v5 จัดการ Server State และ Ant Design สำหรับส่วนติดต่อผู้ใช้ ส่วนหลัง (Backend) พัฒนาด้วย Node.js/Express.js ร่วมกับ Sequelize ORM เชื่อมต่อฐานข้อมูล MySQL 8.0 มีระบบแจ้งเตือนแบบ Real-time ผ่าน Socket.io และการสร้างเอกสาร PDF หนังสือราชการอัตโนมัติด้วย PDFKit รองรับอักขระภาษาไทย ทั้งระบบ Deploy ด้วย Docker Compose บน VPS พร้อม CI/CD ผ่าน GitHub Actions

ผลการพัฒนาได้ระบบที่ครอบคลุม 3 กระบวนการหลัก ได้แก่ (1) การฝึกงาน ตั้งแต่การยื่นคำร้อง คพ.05 การออกหนังสือราชการอัตโนมัติ 3 ประเภท การบันทึกการฝึกงานรายวัน จนถึงการประเมินจากผู้ควบคุมงานผ่านลิงก์อีเมลโดยไม่ต้องสมัครสมาชิก (2) โครงงานพิเศษ ตั้งแต่การเสนอหัวข้อ การสอบหัวข้อ การบันทึกการพบอาจารย์ที่ปรึกษา จนถึงการยื่นคำร้องขอสอบ และ (3) ปริญญานิพนธ์ ตั้งแต่การขอทดสอบระบบ การยื่นสอบปริญญานิพนธ์ จนถึงการบันทึกผลสอบและสถานะเล่ม ระบบรองรับ 4 บทบาทผู้ใช้งาน ได้แก่ นักศึกษา อาจารย์ที่ปรึกษา เจ้าหน้าที่ภาควิชา และผู้ควบคุมงานจากสถานประกอบการ พร้อมทั้งระบบรายงานและส่งออกข้อมูลในรูปแบบ Excel สำหรับเจ้าหน้าที่

ระบบผ่านการทดสอบด้วย Playwright E2E Tests จำนวน 160 กรณีทดสอบ การทดสอบ Backend Integration และการตรวจสอบความปลอดภัย ได้แก่ การแก้ไขช่องโหว่ IDOR 6 จุด การทดสอบสิทธิ์การเข้าถึง 40 เส้นทาง และ RBAC Policy Audit ระบบได้ถูกนำไปใช้งานจริง (Deploy) ที่ https://cslogbook.me โดยมีข้อมูลจริงของนักศึกษาประกอบด้วย โครงงานพิเศษ 41 โครงงาน และการฝึกงาน 90 รายการ ในปีการศึกษา 2567-2568

## เปรียบเทียบเดิม vs ใหม่

| ส่วน | เดิม | ใหม่ |
|-----|-----|-----|
| สรุปงาน (ปัญหา + วัตถุประสงค์) | ✅ มี (2 ย่อหน้า ยาวเกินไป) | ✅ มี (1 ย่อหน้า กระชับ) |
| อ้างอิงหลักสูตร | ❌ แค่กล่าวผ่าน | ✅ ระบุชัด: หลักสูตร พ.ศ. 2564, ปี 2567-2568 |
| เทคโนโลยีที่ใช้ | ❌ ไม่มีเลย | ✅ ครบ: Next.js, React, Node.js, MySQL, Socket.io, PDFKit, Docker |
| ผลการพัฒนา (ฟีเจอร์) | ❌ บอกแค่กว้างๆ | ✅ ระบุ 3 กระบวนการ, 4 roles, PDF 3 ประเภท, Reports + Export |
| ผลการทดสอบ | ❌ ไม่มี | ✅ E2E 160 tests, IDOR 6/6, route-access 40 tests |
| การนำไปใช้งาน | ❌ ไม่มี | ✅ deploy cslogbook.me, ข้อมูลจริง 41 โครงงาน + 90 ฝึกงาน |
