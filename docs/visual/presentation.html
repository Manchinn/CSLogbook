Architecture Overview

Key Features
ฟีเจอร์หลักที่ออกแบบเพื่อทุก role
📚 Logbook & Meeting
บันทึกการพบอาจารย์พร้อม timestamp, date validation, และ teacher approval queue แบบ real-time
📋 Document Pipeline
สร้าง PDF หนังสือราชการ 3 ประเภทอัตโนมัติ ด้วย TH Sarabun New + ตราครุฑ
🔔 Notification & Agents
แจ้งเตือน Email (Gmail REST API) + Socket.io real-time ทุก workflow event พร้อม cron agents
📊 Reports & Admin
Dashboard สถิติ, Document Pipeline report, CSV/XLSX export, Teacher queue summary
🔒 Security & Auth
KMUTNB SSO login, JWT refresh, RBAC middleware, IDOR prevention (6 fixes), Rate limiting
⚙️ Academic Settings
ตั้งค่าปีการศึกษา, กำหนด deadline dynamic, DeadlineTimeline component, ThaiDate input (พ.ศ.)

Gmail REST API
  • Gmail REST API (migration จาก nodemailer)

ปัญหาที่เจอ / อุปสรรคระหว่างการพัฒนา
01Thai Font ใน PDF
❌ pdfkit ไม่รองรับ Thai font → ตัวอักษรไทยไม่แสดง
✅ เพิ่ม TH Sarabun New + Loma font ใน Docker image และ config pdfkit
02IDOR Vulnerability
❌ PDF endpoint ไม่ตรวจ ownership → user A ดู PDF ของ user B ได้
✅ เพิ่ม ownership check ทุก PDF endpoint (6 จุด) + automated security tests
03Workflow State Complexity
❌ State กระจายใน 3 layers ไม่ sync กัน ทำให้เกิด phantom status
✅ สร้าง workflowStates.ts + WORKFLOW_STATES.md audit ทำให้ทุก layer ใช้แหล่งเดียว
04MySQL UTF-8mb4 บน Docker
❌ Thai characters หายเมื่อ import data ผ่าน docker exec ไม่ระบุ charset
✅ เพิ่ม --default-character-set=utf8mb4 ทุกครั้งที่รัน mysql บน production
