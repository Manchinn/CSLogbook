🔐 Authentication Flow

sequenceDiagram
    participant U as 👤 User
    participant F as 💜 Frontend
    participant A as 🔵 Auth API
    participant DB as 🗄️ MySQL
    participant MW as 🛡️ Middleware

    U->>F: Login (Email/Password or SSO)
    F->>A: POST /api/auth/login
    A->>DB: User.findOne({ email, active })
    DB-->>A: User record
    A->>A: Verify password / SSO token
    A-->>F: JWT Token (userId, role, exp)
    F->>F: Store in localStorage

    Note over F: ทุก Request ต่อจากนี้

    F->>MW: API Request + Bearer Token
    MW->>MW: authenticateToken (verify JWT)
    MW->>DB: Lookup User + Student/Teacher
    MW->>MW: authorize(resource, action)
    MW-->>F: 200 OK / 401 / 403

🛡️ Security Measures

Token expiry check ทุกครั้งที่โหลดหน้า  •  Session หมดอายุอัตโนมัติ  •  Rate limiting ป้องกัน brute-force  •  CORS whitelist  •  KMUTNB SSO integration

👥 Role-Based Access Control

🎓

Student

Base Role
  • ส่งเอกสาร / เขียน Logbook
  • สร้างโปรเจกต์ / Milestone
  • ลงทะเบียนฝึกงาน
  • ขอ Email Approval
  • ดูข้อมูลตนเอง
  • KP02 Submit / Topic Exam
👨‍🏫

Teacher

+ Sub-permissions
  • ตรวจเอกสาร / อนุมัติ
  • ดู Logbook / ให้ Approve
  • สอบหัวข้อ / System Test
  • จัดการ Timeline step
  • teacher:type (academic/support)
  • teacher:position:* (หัวหน้าภาค)
  • teacher:topic_exam_access
  • teacher:can_export_project1
🛡️

Admin

Full Access
  • จัดการผู้ใช้ทั้งหมด
  • ตั้งค่า Academic / Curriculum
  • จัดการ Workflow / Deadline
  • ดู Reports ทั้งระบบ
  • Upload CSV / Settings
  • Archive Projects

Permission Resolution Pipeline

buildPrincipalKeys(user) → base role: "student" | "teacher" | "admin" → if teacher: + "teacher:${teacherType}" → if teacher: + "teacher:position:${position}" → if teacher: + flags (topic_exam_access, can_export_project1) → authorize(resource, action) → match principalKeys vs permissions[resource][action] → allow / deny (403)
